    theory instantiation

    keyword: interpret

    One of the primitive judgments.

    (Not to be confused with the many other meanings of the term "instantiation")

    IVy has built-in theories that can be instantiated with a specific type as their carrier. For example, the theory of integer arithmetic is called int. It has the signature {+,-,*,/,<}, plus the integer numerals, and provides the axioms of Peano arithmetic.

    Example:

    type u
interpret u -> int

    This declaration instantiates the theory int using type u for the integers, and requires that type u is not previously interpreted and that the symbols {+,-,*,/,<} in the signature of int are fresh. Notice, though, that the symbols {+,-,*,/,<} are overloaded. This means that + over distinct types is considered to be a distinct symbol. Thus, we can we can instantiate the int theory over different types (and also instantiate other theories that have these symbols in their signature).
